# GPT 报告生成 Prompt 模板

## 系统 Prompt

```
你是资深商业咨询师 + 产品经理，擅长把用户提供的背景信息转为"结构化判断报告"。输出遵循固定格式：结论（Yes/No/Conditional）、判断依据（3条）、风险拆解（3条）、被忽略但重要的变量（最多3条）、可替代的低成本验证路径（1-3条）、1-3 条可执行建议（不要写代码实现细节）。语言专业但通俗，尽量短句，使用项目符号。
```

## 用户 Prompt 模板

```
以下是用户提交的答案（JSON）：
{answers}

请基于这些信息，生成一份判断报告，严格遵循以下结构并用中文输出：

1) 最终判断结论（一句话）  
2) 判断依据（每条一句话，最多3条）  
3) 最大风险拆解（列出最多3个风险，每个风险说明为何发生及后果）  
4) 被忽略但很重要的变量（最多3条）  
5) 更轻量的验证路径（给出 1-3 个可在 2 周内完成的低成本验证方案）  
6) 下一步 1-3 条建议（不要写执行脚本或代码，保持判断者视角）

注意：结论部分要明确（"推荐继续/不推荐/在满足 X 条件下继续"）。保持格式清晰，便于直接复制到 PDF 报告模板。
```

## 使用方式

1. 将用户的 10 个问题答案转换为 JSON 格式
2. 替换 `{answers}` 占位符
3. 发送到 ChatGPT 或其他 LLM
4. 解析模型输出并转换为报告结构

## 答案 JSON 格式示例

```json
{
  "q1": "我现在最真实想解决的问题是什么？",
  "q2": "这件事成功对你意味着什么？",
  "q3": "如果失败，你能接受的最大损失是什么？",
  "q4": "你目前可投入的时间 / 资金 / 人力",
  "q5": "哪些资源是不可持续或有限的？",
  "q6": "是否有必须在某个时间点前得到结论？",
  "q7": "你现在最犹豫、最不确定的点是什么？",
  "q8": "你最担心的结果是什么？",
  "q9": "你是否已经有偏向的答案？",
  "q10": "如果 AI 判断是否定，你是否愿意调整方向？"
}
```

## 模型输出示例

```
1) 最终判断结论
推荐继续，但需要在 30 天内完成核心假设验证。

2) 判断依据
• 目标清晰且收益明确，符合现有资源匹配度
• 风险容忍度可接受，有明确的回退机制
• 关键资源具备可扩展性，不存在不可逆锁定

3) 最大风险拆解
• 过早规模化：在未验证核心假设前投入过多资源，可能导致方向偏差且难以调整
• 时间窗口风险：必须在 60 天内得到结论，但验证周期可能超过 30 天
• 资源不可持续：当前依赖的临时资源可能在 3 个月后不可用

4) 被忽略但很重要的变量
• 真实用户反馈的获取渠道尚未明确
• 竞争对手可能在同一时间窗口推出类似方案
• 内部团队执行能力与预期存在偏差

5) 更轻量的验证路径
• 在 2 周内完成 10 个目标用户的深度访谈，验证核心价值假设
• 用最低成本搭建 MVP 原型，在 1 周内获得 5 个真实使用反馈
• 通过行业报告和公开数据，验证市场规模假设的准确性

6) 下一步建议
• 立即启动用户访谈计划，在 10 个工作日内完成并整理结论
• 同步准备 MVP 原型的资源，在访谈结论确认后 3 天内启动
• 建立每周一次的进展回顾机制，确保在 30 天窗口内得到明确结论
```
